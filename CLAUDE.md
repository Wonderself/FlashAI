## script.js architecture

### Dynamic content (NOT in static HTML)
- **Legal modals** (~line 1779): `mentions-legales`, `politique-confidentialite`, `cgv` — generated by JS, triggered by footer links
- **Chatbot** (~line 736): Trilingual i18n object (`i18n.es`, `i18n.en`, `i18n.fr`) with language selection at start. State var `lang`.

- **Service modal** (`serviceData`): 4 current LATAM services

- **FAQ accordion** (~line 1170): tax, pricing, sovereignty, services categories

### Known technical issues
- **Edit tool + unicode escapes**: `script.js` contains `\u{1F680}`, `é`, etc. The Edit tool often fails to match these. Use Python `open/read/replace/write` pattern instead:
  ```python
  python3 -c "
  f=open('script.js','r'); c=f.read(); f.close()
  c=c.replace('old','new')
  f=open('script.js','w'); f.write(c); f.close()
  "
  ```
- For block replacements in script.js, use marker-based positional replacement (find start marker, find end marker, splice)
